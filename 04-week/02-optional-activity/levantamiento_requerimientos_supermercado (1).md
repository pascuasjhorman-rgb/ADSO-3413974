# Documento de Levantamiento de Requerimientos

## Técnica: Encuesta

Proyecto: Sistema POS y Portal Web para Supermercado Mediano\
Personal del supermercado: 30 empleados\
Operación: Manejo frecuente de promociones

------------------------------------------------------------------------

# 1. Alistamiento

## 1.1 Objetivo de la Encuesta

El objetivo de esta encuesta es recopilar información relevante de los
usuarios internos y externos del supermercado para identificar
necesidades, problemas actuales y expectativas relacionadas con la
implementación de:

-   Un **Sistema POS (Point of Sale)** para la tienda física.
-   Un **Portal Web para clientes** que permita visualizar productos,
    promociones y gestionar un carrito de compras para realizar pedidos.

Los resultados permitirán identificar **requerimientos funcionales y
necesidades de usabilidad** que servirán como base para el diseño del
sistema.

------------------------------------------------------------------------

## 1.2 Herramientas a Utilizar

Para la recolección de información se utilizarán herramientas digitales
que faciliten la distribución, recopilación y análisis de respuestas.

Herramientas propuestas:

-   **Google Forms**: para diseñar y distribuir las encuestas.
-   **Google Sheets**: para análisis preliminar de datos.
-   **Excel o herramientas de análisis de datos**: para generación de
    gráficos y porcentajes.

------------------------------------------------------------------------

------------------------------------------------------------------------

# Análisis de Resultados

Los resultados muestran una **fuerte necesidad de modernización del
sistema de ventas** del supermercado.

En el caso del personal interno, se identificaron problemas
principalmente relacionados con:

-   Lentitud en el registro de ventas
-   Errores en el control de inventario
-   Dificultades para aplicar promociones

Esto indica que el nuevo sistema POS debe priorizar **rapidez,
automatización y facilidad de uso**.

Por parte de los clientes, existe un **alto interés en consultar
promociones online y utilizar herramientas digitales** para planificar
compras.

También se identificó que la mayoría de los clientes utiliza
**dispositivos móviles**, por lo que el portal debe tener un diseño
**responsive**.

# Conclusión Técnica: Requerimientos Identificados

## Requerimientos Clave para el Sistema POS

1.  Registro rápido de ventas mediante interfaz simplificada.
2.  Integración con lectores de código de barras.
3.  Actualización automática del inventario tras cada venta.
4.  Gestión dinámica de promociones y descuentos.
5.  Generación automática de reportes diarios de ventas.
6.  Control de productos agotados en tiempo real.
7.  Interfaz fácil de usar para cajeros y administradores.

## Requerimientos Clave para el Portal Web

1.  Catálogo de productos accesible para los clientes.
2.  Visualización destacada de promociones y ofertas.
3.  Motor de búsqueda de productos.
4.  Funcionalidad de carrito de compras.
5.  Visualización de disponibilidad de productos.
6.  Portal optimizado para dispositivos móviles.
7.  Posibilidad de preparar pedidos antes de ir a la tienda.

Este documento servirá como base para las siguientes fases del proceso
de **ingeniería de requisitos**, incluyendo análisis detallado, modelado
de requerimientos y diseño del sistema.
